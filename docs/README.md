# Docs Guide

Use these as the current source of truth:

- `next_steps.md` for active product and engineering priorities
- `holistic_review.md` for the current technical assessment of the codebase
- `support_matrix.md` for release validation and supported-environment claims

Use these as secondary or historical references:

- `architecture.md` for design rationale and long-term shape (historical; not the current implementation plan)
- `roadmap.md` for high-level product stages (historical; not current priorities)

These are archived and should not be treated as current plans:

- `implementation_plan.md` — superseded by `next_steps.md`
- `ship_todo.md` — superseded by `next_steps.md`
